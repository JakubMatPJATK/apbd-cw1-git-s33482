1. Kiedy Git wykona fast-forward, a kiedy powstaje merge commit?

Git wykonuje Fast-forward, gdy gałąź, którą chce scalić jest gałęzią nastepczą któa scalamy, a merge commit gdy na obu gałęziach doszło do zmian

2. Czym w praktyce różni się merge od rebase?

merge scala zmiany z dwóch gałęzi w jedną a rebase przenosi commity tworzac liniowa historie commitow
